                                                               С чего начать мобильное тестирование.
С чего начать мобильное тестирование? В первую очередь нужно определиться с выбором устройства, а также собрать статистику по мобилке: Выбор ОС, выбор форм-факторов.Информацию можно взять с сайтов для андроид и ios. Второй вопрос будет какого типа будет мобильное приложение? Существует 3 типа: нативное, гибридное, веб. Нативное приложение написано на языках котлин для андроид, свифт для ios. Гибридное написано на одном языке для двух платформ, например JS. Веб приложение - это по сути веб сайт, адаптированный для мобилки. Взаимодействие происходит через браузер. После того как мы определились с типом устройства, на каком языке будет приложение, переходим к следующему требованию - гайдлайны. После изучения гайдланов нужно определиться как тестить приложение. Есть эмуляторы и симуляторы моб. приложений. Эмуляторы стоят на андройд - это специальная программа, в которой можно поработать с приложением. На компьютере поднимается отдельная операционная система. На симуляторах этого не нужно делать. Запуск приложения происходит без использования оперативной системы. Симуляторы стоят на ios.
Особенности мобильного тестирования
                                                                            Прерывания
Прерывания - что может прервать работу нашего приложения.  Что может выскочить при работе приложения, поверх.
1.Звонки на телефон
2.СМС
3.Пуш-уведомления
4.Подключение зарядки
                                                                            Установка
Тестинг установки - проверяем установку/обнову/удаление/переустановку приложения.
1.Установка с 0.
2.Удаление
3.Повторная установка
4.Установить предыдущую версию, удалить предыдущую версию, установить новую версию
5.Установить предыдущую версию и поверх поставить новую(обнова)
6.Установить старую, поверх поставить новую, откат к старой, удостовериться что все раб и поставить новую
7.Установить предыдущую версию, поверх новую,удалить и поставить новую
8.Установить старую версию,удалить, установить новую версию,удалить, установить новую и проверить как работает.
9.Приложение установить как АПК, а потом поверх АПК установить из плеймаркет
10.Поставить из гугл плей, поверх поставить АПК.
                                                                             Реконнект
Проверка работы приложения на различных типах соединения: WI-FI/4g/3g/2g, плохой сигнал, потеря связи,изменение типа соединения.
1.WI-fi перешел на 4/3/2g
2.Полностью откл. wi-fi
3.Мобильные данные полностью отключились
4.Вай-фай перешел на другой вайфай
5.Режим самолет
6.Режим модем
7.Режим не беспокоить
                                                                      Работа с функциями телефона
Проверяем наше приложение на функции телефона( то есть исходя для чего мы делаем наше приложения, те функции и проверяем)
Примеры: GPS,фото/видео,размер экрана/ориентация,работа с жестами, датчики,фонарики,режим звук/вибрация/без звука, подключение наушников по блютуз/через джек, подключение часов,модуль NFC,режим чтения,запись экрана, трансляция экрана, темный/ночной режим,скриншоты.
                                                                     Тестирование производительности
Загрузить оперативную память(включить все приложения), Проверить при низком заряде батареи, при низкой/высокой температуре телефона, загрузка приложения с внешнего/внутреннего накопителя.
